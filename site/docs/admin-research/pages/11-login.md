# 登录页（/login）后台数据调研

## 页面目标
邮箱登录、注册、Google快捷登录。

## 前端需要后台管理的数据
- 认证接口
  - `POST /auth/login`
  - `POST /auth/register`
  - `POST /auth/google`
- 用户扩展资料：country/travelStyle（本地扩展，建议后端落库）

## 后台设计
- 模块：`账号中心`
  - 账号、角色、状态、登录方式
- 模块：`第三方登录配置`
  - Google OAuth 配置与回调
- 模块：`安全策略`
  - 密码规则、登录限流、风控黑名单

## 后台用户流程
1. 用户选择邮箱或Google登录。
2. 后端签发token并返回用户信息。
3. 前端持久化会话并跳转首页。
4. 后台可在账号中心查看用户和状态。
