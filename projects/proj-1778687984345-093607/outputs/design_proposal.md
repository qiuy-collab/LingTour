# 前端重构设计方案：Shopify 级高端旅游/口译平台

## 1. 视觉设计语言 (Shopify 对标)
为了对标 Shopify 独立站的高端感，我们将抛弃繁琐的堆砌，转向“内容为王”的极简主义。

*   **排版 (Typography)**：
    -   标题：使用具备高级感的衬线体（如 *Playfair Display*）或极简的无衬线体（如 *Inter*），加大字间距（Letter Spacing）。
    -   正文：统一使用无衬线体，行高（Line Height）设定在 1.6-1.8 之间，确保阅读呼吸感。
*   **配色 (Color Palette)**：
    -   主色调：象牙白 (#F9F9F9)、炭黑色 (#1A1A1A)。
    -   辅助色：香槟金或莫兰迪灰调，用于强调细节而不破坏整体高级感。
*   **空间 (Spacing)**：
    -   增加区块间的白空间（Whitespace），让内容“呼吸”。
    -   容器圆角统一为极小的 R2-R4，或者完全直角，呈现工业设计感。
*   **动效 (Transitions)**：
    -   淡入淡出 (Fade-in)、平滑滚动 (Smooth Scroll) 和微小的缩放动效。

## 2. 核心模块重构方案

### 2.1 动态事件系统 (Hero & Calendar)
*   **近期路线推荐轮播**：
    -   **设计**：通栏全屏 Hero Section，背景为高质感路线实拍图。
    -   **逻辑**：叠加“倒计时”标签（例如：“离洗夫人文化节还有 3 天”）。
    -   **交互**：自动读取地点元数据中的节日信息，优先展示即将发生的事件。
*   **风俗文化日历**：
    -   **展现**：侧边浮动或底部收纳的 Minimalist Calendar。
    -   **内容**：标注广东地区特色（如：湛江年例、佛山秋色、龙舟节）。点击日期直接关联推荐路线。

### 2.2 地图驱动的路线导航 (Map-First Navigation)
*   **多层级地图下钻**：
    -   **大地图层**：广东省行政区划图，点击特定城市（如：湛江、梅州）产生平滑缩放进入小地图。
    -   **小地图层**：展示地形与细分街道/景点坐标。
*   **路线展示与交互**：
    -   **真实路线渲染**：利用 SVG 或 Mapbox 绘制真实轨迹（Polyline）。
    -   **高亮逻辑**：多条路线共存时，非选中路线为半透明灰色，鼠标悬停或点击后变更为品牌主色并加粗，同时在侧边栏弹出该路线的简要摘要。

### 2.3 节点索引详情页 (Node-based Detail)
*   **交互逻辑**：左侧/顶部保留地图，右侧为内容区。地图上的每个节点（Node）作为导航锚点。
*   **内容结构**：
    -   **节点 (Nodes)**：时间线形式排列。
    -   **故事 (Stories)**：点击节点，内容区优先展示该地的文化背景、民俗故事（以精美的卡片形式展示）。
    -   **安排 (Arrangements)**：包含景点/饭店/酒店。每个地点配备“定位”按钮，点击跳转官网或高德/Google地图。
    -   **联动**：用户在右侧滚动内容时，左侧地图自动平移至对应节点位置。

### 2.4 社交评论系统 (Order-Locked Post)
*   **评论入口安置**：
    -   **路线详情页底部**：作为一个“社区印记”模块，展示真实旅行者的 Post。
    -   **节点级入口**：在每个 Node 详情下，有一个“看看大家在这里说了什么”的小入口。
*   **订单锁定机制**：
    -   用户个人主页设有 Post 发布区。
    -   **逻辑校验**：发布时系统检索用户订单。若未购买该路线，发布按钮置灰，并提示“完成这段旅程后再来分享您的故事吧”。
    -   **透明度**：所有 Post 均贴有“已认证旅行者”标签。

### 2.5 精简口译服务页 (Simplified Interpretation)
*   **场景化套餐**：
    -   不再罗列价格表，改为：商务洽谈、旅游陪同、技术考察、展会服务四种大卡片。
*   **分级选择逻辑**：
    -   Step 1: 选择场景。
    -   Step 2: 选择译员级别（初级/中级/高级，附带译员头像与资历）。
    -   Step 3: 自动计算“组合价格” = (场景基数 + 译员等级加成) * 天数。
*   **设计**：使用 Shopify 常见的侧边抽屉（Drawer）进行预约信息的确认，保持页面主流程简洁。

## 3. 技术实施策略
1.  **前端框架**：Next.js + Tailwind CSS (用于响应式与极简布局)。
2.  **地图技术**：Mapbox GL JS (支持高性能 SVG/Web GL 路线渲染与缩放)。
3.  **状态管理**：Zustand (管理地图节点与内容区的实时联动同步)。
4.  **社交数据**：建立 `Orders` 与 `Posts` 的多对一关联数据库模型。
