# Home 首页用户操作与交互流程

## 1. 页面定位

首页不再是“平台介绍页”，而是一个 **灵感触发页 + 转化入口页**。

用户进入首页后，应该在 10 秒内理解三件事：

1. 这里可以发现广东文化路线；
2. 近期有哪些节庆 / 民俗 / 活动值得出发；
3. 可以继续进入路线、文化、口译或旅行者手记。

首页需要对标 Shopify 高端独立站：首页不是把所有内容讲完，而是用强视觉、少量文案和清晰 CTA 引导用户继续探索。

---

## 2. 用户进入首页后的第一屏

### 用户看到

- 大幅 Hero 图 / 视频；
- 一句强主张；
- 一句辅助说明；
- 1-2 个核心 CTA。

### 推荐信息结构

```text
Hero 主标题：
Explore Guangdong Through Living Culture

副标题：
从节庆、非遗、烟火市集与海岸村落，进入一条真正有故事的广东路线。

CTA：
[探索路线] [查看近期活动]
```

### 用户操作

1. 点击「探索路线」进入 `/routes`；
2. 点击「查看近期活动」滚动到事件路线轮播 / 活动日历；
3. 向下滚动进入首页核心内容模块。

---

## 3. 近期事件路线推荐轮播

### 目的

把“近期即将发生的节庆 / 风俗 / 市集 / 演出 / 美食季”转化为路线推荐。

例如：

```text
距离湛江年例还有 12 天
推荐路线：湛江年例文化路线
```

### 用户看到

轮播卡片包含：

- 活动名称；
- 活动地区；
- 倒计时：还有几天 / 几周；
- 关联路线名称；
- 1 张高质量图；
- 简短推荐理由；
- CTA：查看路线。

### 用户操作

1. 左右切换轮播；
2. hover / focus 某张卡片时图片轻微放大，显示路线标签；
3. 点击卡片或 CTA 进入对应路线详情；
4. 若一个活动关联多条路线，点击后进入 `/routes?event=xxx` 的筛选结果。

### 状态设计

- **有近期活动**：按时间临近程度排序；
- **没有近期活动**：展示“本月推荐文化路线”；
- **活动已结束**：自动降权或显示“回顾 / 明年提醒”。

---

## 4. 广东活动日历

### 目的

让用户按时间浏览广东地区的风俗节日、地方活动和路线机会。

### 用户看到

```text
广东活动日历
[五月 2026]

分类筛选：
全部 / 节庆 / 民俗 / 市集 / 表演 / 美食 / 季节限定

左侧：月历
右侧：当月活动列表
```

### 用户操作

1. 切换月份；
2. 点击分类筛选；
3. 点击某一天查看当天活动；
4. 点击活动卡片展开详情；
5. 点击「查看相关路线」进入路线详情或路线筛选页。

### 活动详情内容

- 活动名称；
- 时间范围；
- 城市 / 地区；
- 分类；
- 简介；
- 关联路线；
- 推荐出行时间；
- CTA：查看路线 / 收藏活动。

---

## 5. 首页轻量社区入口

### 目的

用真实用户内容提高信任感，但不让社区内容抢占首页主线。

### 用户看到

```text
旅行者手记 Traveler Notes
看看走过这些路线的人怎么说
```

展示 3-6 条精选 post：

- 用户头像 / 名称；
- post 标题；
- 关联路线；
- 一张图；
- 简短摘录。

### 用户操作

1. 点击 post 进入 post 详情或对应路线详情下的 post 锚点；
2. 点击「查看全部手记」进入 `/posts`；
3. 点击关联路线进入 `/routes/[slug]`。

## 6. 首页转化路径

```mermaid
flowchart TD
    A[用户进入首页] --> B[看到 Hero 主视觉]
    B --> C{用户兴趣点}
    C -->|想找路线| D[点击探索路线]
    C -->|被近期活动吸引| E[浏览事件路线轮播]
    C -->|按时间计划旅行| F[查看广东活动日历]
    C -->|想看真实反馈| G[浏览旅行者手记]
    D --> H[/routes 地图探索页]
    E --> I[/routes/[slug] 路线详情]
    F --> J[/routes?event=xxx 活动相关路线]
    G --> K[/posts 旅行者手记]
    I --> L[预订路线 / 预约口译]
```

## 8. 首页交互重点

- 第一屏不要堆太多平台介绍；
- 近期活动要有倒计时，制造行动理由；
- 活动日历要服务路线转化，不只是信息展示；
- post 区只做信任背书，更多内容放到 `/posts`；
- CTA 文案要明确：查看路线、查看活动、预约口译、阅读手记。
