# Culture 文化页用户操作与交互流程

## 1. 页面定位

Culture 页面不是单纯的文化百科页，而是 **文化主题入口 + 路线种草页**。

它需要解决的问题是：

1. 用户不一定知道自己想去哪里，但可能对某种文化主题感兴趣；
2. 通过文化主题把用户引导到路线、活动和故事；
3. 用更轻、更高级的方式展示广东地域文化，而不是大段知识堆叠。

推荐中文命名可选：

- 文化灵感；
- 岭南文化；
- Culture；
- Discover Culture。

---

## 2. 用户进入 Culture 页后的第一屏

### 用户看到

- 大图 / 视频 Hero；
- 一句文化主题主张；
- 少量文化关键词；
- CTA：探索文化主题 / 查看相关路线。

### 推荐信息结构

```text
岭南不是一种风格，而是一组正在发生的生活方式

从广府骑楼、潮汕工夫茶、客家围屋、雷州年例到海岛渔市，
用路线进入广东的文化现场。

CTA：
[探索文化主题] [查看文化路线]
```

---

## 3. 文化主题卡片

### 目的

把文化内容产品化，避免百科式长文。

### 推荐主题

- 广府文化；
- 潮汕文化；
- 客家文化；
- 雷州 / 粤西民俗；
- 非遗手作；
- 海岛与渔港；
- 美食与市集；
- 节庆与仪式。

### 用户看到

每个主题卡片包含：

```text
主题名
一句话说明
代表地区
关联路线数量
近期活动数量
CTA：查看主题
```

### 用户操作

1. hover 卡片：图片轻微放大，显示代表城市；
2. 点击卡片：进入主题详情或展开主题内容；
3. 点击「查看相关路线」进入 `/routes?culture=xxx`；
4. 点击「查看近期活动」跳到活动日历筛选结果。

---

## 4. 文化主题详情

### 页面 / 模块结构

当用户选择一个文化主题，例如“潮汕文化”，展示：

```text
潮汕文化
一句话导语

核心体验：
- 工夫茶
- 祠堂与街巷
- 潮剧 / 英歌舞
- 牛肉火锅 / 粿品

相关城市：
汕头 / 潮州 / 揭阳

推荐路线：
潮汕工夫茶与古城路线

近期活动：
某某民俗活动 / 某某市集
```

### 用户操作

1. 浏览文化故事摘要；
2. 点击核心体验标签筛选路线；
3. 点击城市进入该地区路线；
4. 点击推荐路线进入路线详情；
5. 点击活动进入活动详情或关联路线。

---

## 5. 地区文化索引

### 目的

让用户从“地区”而不是“主题”进入文化路线。

### 用户看到

可以用小地图或区域卡片展示：

- 广州 / 佛山 / 珠三角；
- 潮州 / 汕头 / 揭阳；
- 梅州 / 河源；
- 湛江 / 茂名 / 阳江；
- 韶关 / 清远等。

### 用户操作

1. hover 地区：显示代表文化和路线数量；
2. 点击地区：进入 `/routes?region=xxx`；
3. 或在当前页更新文化主题列表。

---

## 6. Culture 与活动日历的关系

Culture 页应该能承接首页的广东活动日历。

例如用户点击首页日历里的“雷州年例”，可以进入：

```text
/culture?theme=yuexi-folk&event=nianli
```

然后看到：

- 年例是什么；
- 发生在哪些地区；
- 推荐路线；
- 相关 post；
- 最近活动时间。

---

## 7. Culture 与路线的转化关系

Culture 页不能停留在知识阅读，必须导向路线。

### 推荐转化模块

```text
把这段文化放进真实路线里

推荐路线：
- 湛江年例文化路线
- 潮汕工夫茶与古城路线
- 客家围屋慢游路线

CTA：查看路线
```

### 用户路径

```mermaid
flowchart TD
    A[用户进入 Culture 页] --> B[浏览文化主题]
    B --> C{用户选择方式}
    C -->|按主题| D[点击文化主题卡片]
    C -->|按地区| E[点击地区文化索引]
    C -->|按活动| F[点击近期文化活动]
    D --> G[查看主题详情]
    E --> H[查看地区相关文化]
    F --> I[查看活动相关文化]
    G --> J[推荐相关路线]
    H --> J
    I --> J
    J --> K[/routes?culture=xxx]
    K --> L[/routes/[slug] 路线详情]
```

---

## 8. Posts 在 Culture 页的安置

Culture 页可以加入轻量 post 区，但不建议做成评论页。

### 推荐模块

```text
旅行者如何理解这段文化
```

展示与该主题相关的精选 post：

- 用户标题；
- 关联路线；
- 文化标签；
- 摘录；
- 图片。

点击后进入 `/posts/[id]` 或对应路线详情。

---

## 9. 交互重点

- Culture 页不要做成长百科；
- 主题卡片要像产品集合页一样清晰；
- 每个文化主题都要能导向路线；
- 地区、主题、活动三种入口要互相打通；
- post 只做真实体验补充，不抢主题内容主线。
